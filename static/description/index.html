Feature documentation

A closer look at the five areas that do the most work.

01Its own AI, backed by the provider you pick

The agent itself runs inside Odoo — it isn't handed off to some other app. Each agent just calls out to whichever provider's model you assign it, and that choice can be changed per agent without touching its tools, prompts, or conversation history.

  • Anthropic, OpenAI, Gemini, Grok/xAI, and OpenCode — called directly over HTTPS, no vendor SDK required.
  • Ollama for a fully offline, self-hosted model with no API key and no cloud dependency.
  • Per-agent model selection, so different agents can run different providers side by side.
02No-code tool registry

A tool is anything an agent is allowed to call — registered from the UI, no Python required.

  • Odoo ORM actions — search, read, create, update, or run reports against any model.
  • HTTP APIs — call any external REST endpoint as a tool.
  • External MCP servers — pull in tools from any Model Context Protocol–compatible service.
03Access control & rate limiting

Access rules decide what each user group can reach, before a single tool call is made.

  • Restrict by tool, by model, and by agent — independently, per group.
  • Per-group rate limits stop any one user or agent from monopolizing a provider.
  • Sessions default to private, with an explicit toggle to share one with a wider group.
04Session audit, tokens & cost

Every conversation keeps a full, searchable record of what actually happened.

  • Every tool call an agent makes is logged against its session.
  • Token usage and cost are tracked per message, per session, per user.
  • Full transcripts are exportable for review or compliance.
05Optional: MCP server + MCP client

A separate, optional layer on top of the chat feature above — for teams already working with the Model Context Protocol.

  • As an MCP server, it makes your registered Odoo tools callable from other MCP-compatible AI apps.
  • As an MCP client, it can pull in tools from any external MCP server you configure — third-party WhatsApp or YouTube MCP servers are common examples, not something bundled with this module.
  • Same access rules and audit trail apply either way.

Frequently asked

Which AI providers are supported?

Anthropic, OpenAI, Gemini, Ollama, Grok (xAI), and OpenCode. Switch providers per agent without losing its tools, prompts, or conversation history.

Do I need an API key?

Yes, for any hosted provider (Anthropic, OpenAI, Gemini, Grok, OpenCode). Ollama runs fully offline — no key, no cloud dependency.

Can I control what the AI is allowed to do?

Yes. Access rules restrict exactly which tools, models, and agents each user group can reach, with per-group rate limits on top.

Is every action logged?

Every tool call, token count, and cost is recorded per session — searchable per user and per agent.

Does it work on Odoo Online and Odoo.sh?

Yes — Odoo Online, Odoo.sh, and On-Premise installs are all supported.
